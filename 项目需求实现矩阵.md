# AI Agent Framework v3 — 项目需求实现矩阵

> **文档说明**：本文档基于《AI Agent Framework — 开发实现文档 v3》提炼，用于跟踪项目从 0 到 1 的开发进度，确保开发过程严格遵循架构边界和约束。

| :--- | :--- | :--- | :--- | :--- | :--- |
| `INF-01` | 配置管理 | 顶层配置树解析与本地加载 | `FrameworkConfig` (基于 pydantic-settings) | 纯本地读取，按模块(model, context, memory等)拆分结构。 | 待实现 |
| `INF-02` | 日志系统 | 结构化、JSON格式的本地日志 | `StructLogger` (基于 structlog) | 仅用于观测审计，**禁止**业务逻辑依赖日志传递状态。 | 待实现 |
| `INF-03` | 事件总线 | 解耦的内部消息通知机制 | `EventBus` (基于 blinker) | 进程级单例，负责广播如 `run.started`, `tool.completed` 等。 | 待实现 |

## 特殊约束验收清单 (Guardrails)
开发与测试过程中必须 100% 遵守以下红线约束：

- [ ] **依赖传递红线**：底层组件只接收所需最小接口，不得下传 `AgentRuntimeDeps` 上帝对象。
- [ ] **空值语义红线**：失败禁止返回 `None`，必须返回 `Error` 对象/码；空集返回 `[]`。
- [ ] **业务状态红线**：严禁利用日志内容 (`structlog`) 做业务逻辑分支判断。
- [ ] **日志/输出红线**：ToolResult 的 `output` 必须是 JSON 可序列化数据，禁传原始连接/异常对象。
- [ ] **Schema 只读红线**：`ToolMeta.parameters_schema` 在程序运行时绝对不可修改。
- [ ] **状态持有红线**：进程级单例中（如 Config, GlobalToolCatalog）绝对不包含任何单次 Run 的状态数据。
